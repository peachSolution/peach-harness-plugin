# 파일 업로드 가이드

Backend에서 지원하는 파일 업로드 옵션 설명

---

## 파일 업로드 옵션

PRD 작성 시 4가지 옵션 중 선택:

| 옵션 | 설명 | 적용 대상 |
|------|------|----------|
| **1. 필요 없음** | 파일 업로드 기능 없음 | 파일 불필요한 모듈 |
| **2. 일반 파일만** | PDF, DOCX, XLSX 등 | 문서 첨부 필요 |
| **3. 이미지만** | PNG, JPG, GIF 등 | 썸네일, 사진 업로드 |
| **4. 일반 파일 + 이미지** | 둘 다 지원 | 문서 + 이미지 모두 필요 |

**참고**: 모든 파일은 common_file 테이블에 저장되며, parent_code로 연결

## 파일 저장 방식

### Local vs S3 비교

| 구분 | Local | S3 |
|------|-------|-----|
| **저장 위치** | 서버 로컬 디스크 | AWS S3 클라우드 |
| **설정 난이도** | 쉬움 | 보통 (AWS 설정 필요) |
| **확장성** | 제한적 (서버 용량 의존) | 무제한 |
| **비용** | 무료 (서버 스토리지) | 유료 (저장량/전송량 기반) |
| **CDN 연동** | 불가 | 가능 (CloudFront) |
| **서버 독립성** | 서버 종속 | 서버 독립 |
| **백업/복구** | 수동 | 자동 |

### Local 저장

**사용 시기:**
- 개발/테스트 환경
- 소량 파일 (<1GB)
- 단일 서버 환경
- 빠른 프로토타이핑

**특징:**
- 파일 위치: `api/upload/[모듈명]/`
- 간단한 설정, 추가 비용 없음
- 서버 이전 시 파일 마이그레이션 필요

---

### S3 저장

**사용 시기:**
- 프로덕션 환경
- 대용량 파일 (>1GB)
- 이미지 많은 서비스
- 멀티 서버 환경 (로드밸런싱)

**특징:**
- AWS S3 버킷 사용
- CloudFront CDN으로 빠른 다운로드
- 무한 확장성, 서버 독립적
- 자동 백업/복구

---

### 환경별 전환

**특징:**
- 개발: Local, 프로덕션: S3
- 환경 변수로 전환: `FILE_STORAGE=local|s3`
- 코드 수정 없이 배포 전략 변경 가능

**사용 시기:**
- 유연한 배포 전략 필요
- 개발 속도와 프로덕션 안정성 모두 중요

## PRD 작성 예제

### 파일 없음
```markdown
### 파일 기능
| 기능 | 적용 |
|------|------|
| 일반 파일 | N |
| 이미지 | N |
| 저장 방식 | - |
```

### 일반 파일만 (Local)
```markdown
### 파일 기능
| 기능 | 적용 |
|------|------|
| 일반 파일 | Y |
| 이미지 | N |
| 저장 방식 | Local |
```

### 이미지만 (S3)
```markdown
### 파일 기능
| 기능 | 적용 |
|------|------|
| 일반 파일 | N |
| 이미지 | Y |
| 저장 방식 | S3 |
```

### 일반 파일 + 이미지 (S3)
```markdown
### 파일 기능
| 기능 | 적용 |
|------|------|
| 일반 파일 | Y |
| 이미지 | Y |
| 저장 방식 | S3 |
```
