# UI 패턴 가이드

Frontend UI 구현 시 선택할 수 있는 5가지 표준 패턴

---

## 패턴 개요

| 패턴 | 설명 | 사용 시기 |
|------|------|----------|
| **crud** | 목록 + 모달 | 일반적인 CRUD, 입력 10개 미만 |
| **two-depth** | 좌우 분할 (목록/상세) | 목록과 상세를 동시에 표시 |
| **select-list** | 선택 모달 | 다른 화면에서 데이터 참조 선택 |
| **page** | 목록 + 별도 페이지 | 입력 10개 이상, URL 공유 필요 |
| **excel** | 엑셀 업로드/다운로드 | 대량 데이터 입력/추출 |

---

## 1. crud (기본 CRUD + Modal)

### 설명
가장 일반적인 관리자 화면 패턴. 목록 화면에서 모달로 등록/수정

### 사용 시기
- 입력 필드 10개 미만
- 간단한 등록/수정 폼
- 목록 화면에서 빠른 입력

### 특징
- **Modal 사용**: 등록/수정 시 모달 팝업
- **빠른 입력**: 목록 화면에서 바로 등록/수정
- **컴팩트**: 화면 전환 없이 작업 완료

### 화면 구성
- **목록 화면**: 검색 영역 + 테이블 + 페이지네이션
- **등록/수정**: Modal 팝업

### 적합한 예제
- 공지사항 게시판
- 코드 관리
- FAQ 관리
- 간단한 설정

---

## 2. two-depth (투뎁스)

### 설명
화면을 좌우로 분할하여 목록과 상세를 동시에 표시

### 사용 시기
- 목록과 상세를 동시에 보면서 작업
- 빠른 네비게이션 필요
- 마스터-디테일 관계

### 특징
- **좌우 분할**: 목록(좌) + 상세(우)
- **즉시 표시**: 목록 클릭 시 우측에 바로 상세 표시
- **빠른 탐색**: 화면 전환 없이 여러 항목 확인

### 화면 구성
- **좌측**: 검색 영역 + 목록 테이블 + 페이지네이션
- **우측**: 선택된 항목 상세 정보 + 관리 버튼

### 적합한 예제
- 이메일 클라이언트
- 회원 관리 + 상세 정보
- 메시지함
- 파일 탐색기

---

## 3. select-list (선택 모달)

### 설명
다른 화면에서 데이터를 선택하기 위한 모달 리스트

### 사용 시기
- 참조 데이터 선택
- 다대다 관계 설정
- 팝업으로 데이터 선택 필요

### 특징
- **모달 팝업**: 별도 창으로 선택 UI
- **다중 선택**: 체크박스로 여러 항목 선택
- **검색 지원**: 많은 데이터에서 빠르게 찾기

### 화면 구성
- **메인 화면**: 선택 버튼
- **선택 모달**: 검색 + 체크박스 테이블 + 선택 완료 버튼

### 적합한 예제
- 회원 그룹 선택
- 카테고리 선택
- 태그 선택
- 권한 부여

---

## 4. page (페이지 전환)

### 설명
목록과 상세를 별도 페이지(URL)로 분리

### 사용 시기
- 입력 필드 10개 이상
- URL 공유 필요
- 복잡한 폼 (탭, 여러 섹션)
- SEO 필요

### 특징
- **URL 분리**: 각 화면이 고유 URL
- **브라우저 히스토리**: 뒤로가기/앞으로가기 지원
- **북마크 가능**: URL 공유 및 저장
- **큰 폼**: 많은 필드를 여유있게 배치

### 화면 구성
- **목록 페이지**: /entity/list
- **상세 페이지**: /entity/:seq
- **등록/수정 페이지**: /entity/new, /entity/:seq/edit

### 적합한 예제
- 상품 관리 (상세 정보 많음)
- 게시글 작성 (에디터 + 첨부파일)
- 설정 페이지 (여러 섹션)
- 신청서 작성

---

## 5. excel (엑셀 업로드/다운로드)

### 설명
엑셀 파일 업로드/다운로드 기능 포함

### 사용 시기
- 대량 데이터 입력
- 데이터 추출 (통계, 보고서)
- 외부 데이터 임포트

### 특징
- **대량 처리**: 수백~수천 건 데이터 입력
- **템플릿**: 표준 엑셀 양식 제공
- **검증**: 업로드 시 데이터 유효성 검사
- **진행 표시**: 프로그레스 바로 진행 상황 표시

### 화면 구성
- **목록 화면**: 일반 CRUD + [엑셀 업로드][엑셀 다운로드] 버튼
- **엑셀 업로드 모달**: 템플릿 다운로드 + 파일 선택 + 업로드 + 결과 표시

### 적합한 예제
- 상품 대량 등록
- 회원 데이터 임포트
- 통계 데이터 추출
- 주문 내역 다운로드

## 패턴 조합

여러 패턴을 조합하여 사용 가능:

| 조합 | 설명 | 예제 |
|------|------|------|
| crud + excel | 기본 CRUD + 엑셀 업로드/다운로드 | 상품 관리 |
| crud + file | 기본 CRUD + 파일 첨부 | 공지사항 게시판 |
| page + file | 페이지 전환 + 파일 첨부 | 상세한 게시글 작성 |
| two-depth + file | 투뎁스 + 파일 첨부 | 파일 첨부 가능한 메시지함 |

---

## 패턴 선택 가이드

| 조건 | 권장 패턴 |
|------|----------|
| 입력 필드 < 10개 | crud |
| 입력 필드 >= 10개 | page |
| 목록/상세 동시 보기 | two-depth |
| 참조 데이터 선택 | select-list |
| 대량 데이터 입출력 | crud + excel 또는 page + excel |
| URL 공유 필요 | page |
| 빠른 입력 중요 | crud |
| 복잡한 폼 (탭, 섹션) | page |

## PRD 작성 예제

### 기본 CRUD
```markdown
## UI 구성

### 패턴: 기본 CRUD (Modal)

### 화면 구성
- 목록 화면: 검색 + 테이블 + 페이지네이션
- 등록/수정: Modal 팝업
- 필드: 제목, 내용, 공지여부, 사용여부

### 검증
- 제목: 필수, 200자 이내
- 내용: 필수
```

### Two-Depth
```markdown
## UI 구성

### 패턴: Two-Depth

### 화면 구성
- 좌측: 검색 + 목록 테이블 + 페이지네이션
- 우측: 선택된 항목 상세 정보 + 수정/삭제 버튼
```

### Page + Excel
```markdown
## UI 구성

### 패턴: Page 전환 + Excel

### 화면 구성
- 목록 페이지: 검색 + 테이블 + [등록][엑셀 업로드][엑셀 다운로드]
- 상세 페이지: 전체 정보 표시 (URL: /entity/:seq)
- 등록/수정 페이지: 복잡한 폼 (URL: /entity/new, /entity/:seq/edit)
```
