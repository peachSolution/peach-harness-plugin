# 에이전트 팀 설정 가이드

> 작성일: 2026-03-15
> 참고: [Claude Code 공식 문서 — 에이전트 팀](https://code.claude.com/docs/ko/agent-teams)

peach-agent-team / peach-agent-team-refactor 스킬을 사용하려면 Claude Code 에이전트 팀 기능을 활성화해야 합니다.

---

## 1. 사전 요구사항

| 항목 | 요구사항 |
|------|---------|
| Claude Code 버전 | **v2.1.32 이상** (`claude --version`으로 확인) |
| tmux | 분할 창 모드 사용 시 필요 |
| 실험적 기능 플래그 | `CLAUDE_CODE_EXPERIMENTAL_AGENT_TEAMS=1` |

---

## 2. 실험적 기능 활성화

에이전트 팀은 기본적으로 **비활성화** 상태입니다. `~/.claude/settings.json`에 추가하여 활성화합니다:

```json
{
  "env": {
    "CLAUDE_CODE_EXPERIMENTAL_AGENT_TEAMS": "1"
  }
}
```

설정 후 Claude Code를 **재시작**합니다.

### 활성화 확인

```bash
cat ~/.claude/settings.json | grep -i "CLAUDE_CODE_EXPERIMENTAL_AGENT_TEAMS"
# 출력: "CLAUDE_CODE_EXPERIMENTAL_AGENT_TEAMS": "1"
```

---

## 3. tmux 설정 (선택사항 — 분할 창 모드)

에이전트 팀은 두 가지 표시 모드를 지원합니다. **tmux 없이도 에이전트 팀은 정상 동작합니다.**

| 모드 | 설명 | 요구사항 |
|------|------|---------|
| `in-process` (기본) | 메인 터미널 내에서 팀원 순환 (Shift+Down) | 없음 — 모든 터미널에서 작동 |
| `tmux` | 각 팀원이 독립 분할 창 — 동시에 모든 팀원 출력 확인 가능 | tmux 또는 iTerm2 |

> **기본값 `auto` 동작**: 이미 tmux 세션 안에서 실행 중이면 분할 창 모드, 아니면 in-process 모드로 자동 전환됩니다. **메인 대화창을 tmux 안에서 열 필요는 없습니다.**

### tmux 설치

**macOS (Homebrew)**

```bash
brew install tmux
```

**Linux**

```bash
# Ubuntu / Debian
sudo apt install tmux

# Fedora / RHEL
sudo dnf install tmux
```

**설치 확인**

```bash
tmux -V
# 출력 예: tmux 3.3a
```

### settings.json에서 tmux 모드 설정

```json
{
  "env": {
    "CLAUDE_CODE_EXPERIMENTAL_AGENT_TEAMS": "1"
  },
  "teammateMode": "tmux"
}
```

> **기본값(`auto`)**: 이미 tmux 세션 내에서 실행 중이면 분할 창, 아니면 in-process.

### tmux 세션에서 Claude Code 실행 (분할 창 모드 원할 때)

분할 창으로 모든 팀원을 동시에 보고 싶다면 tmux 세션 안에서 실행합니다:

```bash
# 새 tmux 세션 시작
tmux new-session -s dev

# 세션 내에서 Claude Code 실행 → 자동으로 분할 창 모드 적용
claude

# tmux 세션 분리 (백그라운드 유지)
Ctrl+B, D

# 세션 재연결
tmux attach -t dev
```

> **iTerm2 사용자**: `tmux -CC`로 실행하는 것이 권장 방법입니다.
> iTerm2 → Settings → General → Magic → **Enable Python API** 활성화 필요.

### tmux 주요 단축키

| 단축키 | 동작 |
|--------|------|
| `Ctrl+B, "` | 수평 분할 |
| `Ctrl+B, %` | 수직 분할 |
| `Ctrl+B, 방향키` | 창 이동 |
| `Ctrl+B, D` | 세션 분리 |

## 4. 에이전트 팀 조작 단축키

| 단축키 | 동작 |
|--------|------|
| `Shift+Down` | 팀원 순환 (in-process 모드) |
| `Enter` | 팀원 세션 보기 |
| `Escape` | 현재 턴 중단 |
| `Ctrl+T` | 작업 목록 토글 |

## 5. peach-agent-team 스킬 실행 전 체크리스트

스킬 실행 시 자동으로 아래 항목을 검증합니다:

1. ✅ `CLAUDE_CODE_EXPERIMENTAL_AGENT_TEAMS=1` 설정 여부
2. ✅ Claude Code 버전 v2.1.32 이상
3. ✅ mode / 모듈명 입력 여부

환경 변수가 미설정된 경우 스킬이 다음 안내를 출력합니다:

```
⚠️  에이전트 팀 기능이 비활성화되어 있습니다.

활성화 방법:
~/.claude/settings.json에 아래 내용을 추가하세요:

{
  "env": {
    "CLAUDE_CODE_EXPERIMENTAL_AGENT_TEAMS": "1"
  }
}

설정 후 Claude Code를 재시작하세요.
```

## 6. 알려진 제한 사항

공식 문서 기준 현재 제한 사항입니다:

- **세션 재개 없음**: `/resume`, `/rewind`는 in-process 팀원을 복원하지 않음
- **세션당 한 팀**: 새 팀 시작 전 현재 팀 정리 필요
- **중첩 팀 없음**: 팀원은 자신의 팀/팀원 생성 불가
- **권한 상속**: 팀원은 리더의 권한 모드를 그대로 상속
- **분할 창 미지원 터미널**: VS Code 통합 터미널, Windows Terminal, Ghostty

---

## 7. 고아 tmux 세션 정리

팀 작업 후 tmux 세션이 남아있는 경우:

```bash
# 세션 목록 확인
tmux ls

# 특정 세션 종료
tmux kill-session -t <session-name>

# 전체 세션 종료
tmux kill-server
```
